# 原始清单与系统字段映射表（v2.0）

> 目标：明确 CSV 原始字段与系统使用字段的对应关系，统一“同名直传/重命名/派生/忽略”四类策略，保障前后端与数据处理的稳定性与可追溯性。

## 映射原则

- 同名直传：系统直接使用原始字段名及取值。
- 重命名：系统使用更规范的内部键名（当前版本尽量避免）。
- 派生：由原始字段通过规则计算出的系统字段（如“电销”“商业险占比”等）。
- 忽略/隐藏：原始字段存在但不在主界面展示（如超长明细“精友车型名称”）。

## 派生字段（系统）

- premium：由`签单/批改保费`汇总得到（净额规则见API）。
- count：由`签单数量`汇总得到。
- telesales_flag：`终端来源 == 0110融合销售`。
- commercial_flag：`险种名称 ∈ {0312,0313,0317}` 可配置。
- new_energy_flag / transfer_flag / non_local_flag：分别对应`是否新能源/是否过户车/是否异地车`为“是”。
- ratios.*：占比类指标（商业险/新能源/过户/异地/电销等），见 `docs/ARCHITECTURE.md` API章节与设计更新说明。

## 维护建议

- 映射以“同名直传”为主，尽量避免重命名；确需重命名时在此表记录并在API契约中说明。
- 脱敏字段在前端只做展示必要裁剪，后端不返回不必要的敏感明细。
- 大型枚举明细（如精友车型名称）在主文档隐藏，仅保留统计；明细移至归档或独立清单。
